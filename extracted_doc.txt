Project Title: Job Portal Management System

1]FUNCTIONAL REQUIREMENTS
FR-1: User Registration
The system shall allow users to register an account using email and password.
Users can register as:
Job Seeker
Recruiter/Employer
The system shall validate user inputs before account creation.

FR-2: User Authentication
The system shall allow registered users to log in using secure authentication.
Authentication shall be handled using JWT tokens.
Users shall remain authenticated across sessions.

FR-3: User Profile Management
Job seekers shall be able to manage their profile information including:
Personal details
Skills
Work experience
Education
Resume upload
Profile picture

FR-4: Company Registration
Recruiters shall be able to create and manage company profiles.
Company profiles shall include:
Company name
Company description
Industry
Website
Location
Logo

FR-5: Job Posting
Recruiters shall be able to create job postings including:
Job title
Job description
Required skills
Salary range
Job location
Job type
Experience level

FR-6: Job Browsing and Search
Job seekers shall be able to browse available job listings.
The system shall allow filtering by:
Job title
Location
Salary
Job type
Experience level

FR-7: Job Application
Job seekers shall be able to apply for jobs using their profile and resume.
The system shall store application data in the database.

FR-8: Application Tracking
Users shall be able to view the status of their job applications.
Application statuses may include:
Applied
Under Review
Shortlisted
Rejected
Accepted

FR-9: Employer Dashboard
Recruiters shall have a dashboard where they can:
Manage job postings
View job applicants
Update application status
Delete job listings

FR-10: Saved Jobs
Job seekers shall be able to bookmark jobs to view later.
Saved jobs shall be stored in the database.

FR-11: Admin Management
Admin users shall be able to:
View all users
Manage job listings
Manage companies
Monitor job applications

2] NON-FUNCTIONAL REQUIREMENTS
1: Security
User credentials shall be securely stored using encrypted passwords.
Authentication shall be protected using JWT tokens.
2: Performance
The system shall provide quick responses when retrieving job listings and user data.

3: Scalability
The system shall support multiple concurrent users without affecting performance.

4: Reliability
All user data including job listings and applications shall be stored reliably.

5: Maintainability
The backend shall follow MVC architecture to ensure maintainability.

6: Availability
The system shall be accessible online via web browser at all times except during maintenance.

User Registration / Login        ↓User Completes Profile        ↓Recruiter Creates Company Profile        ↓Recruiter Posts Job Listings        ↓Job Seekers Browse Jobs        ↓User Applies for Job        ↓Application Stored in Database        ↓Recruiter Reviews Applications        ↓Recruiter Updates Application Status        ↓User Tracks Application Status
7] BACKEND ROUTES DEFINITION
The backend of the Job Portal Management System follows a RESTful API architecture where each resource such as users, jobs, companies, and applications has dedicated routes. These routes allow the frontend to interact with the backend server and perform operations such as creating, retrieving, updating, and deleting data.
